FP-Growth算法

By Frank

发表于 2015-03-22

介绍

    FP-Growth算法也是用于寻找数据集的频繁项集，基于Apriori构建。那已经有了Apriori算法，为什么还需要FP-Growth算法呢？因为效率。在应用Apriori算法的时候，对每一个项集，都需要扫描一遍总数据集，才能判断该项集是否满足最小支持度，当总数据集特别大的时候，Apriori算法的效率是很低的。而FP-Growth算法则不同，它只要扫描两次数据集，然后构建出FP树，再对FP树进行挖掘即可。可以简单的理解为，把数据集的数据信息都压缩到FP树这个结构当中。这篇文章主要分以下几个点。

1.FP树的构建
2.获取条件模式基
3.构建FP条件树并获取频繁项集
4.FPGrowth应用实例
5.完整代码

Apriori算法

By Frank

发表于 2015-03-18

    从这篇开始，把学过的知识点，零散的笔记整理归纳起来，帮助自己巩固一下。没有一定的顺序，但每一篇都力求能够说的明白。

介绍

    Apriori算法属于是数据挖掘中用来寻找频繁项集与进行关联分析的。频繁项集与关联分析又有什么用呢？举一个常见的例子，有超市的购物清单如下。

1.牛奶，面包，水果，饼干
2.牛奶，花生，面包
3.啤酒，牛奶，花生
4.花生，面包，水果
5.啤酒，花生，水果


    找出频繁项集就是要找出购物清单里面经常出现的集合，集合可以由1个或多个事件组成。是否经常出现是人为定义的称为支持度的数据所决定的。而关联分析则是用来寻找频繁项集之间的关系。比如买了牛奶再买面包的可能性？买了啤酒再买花生的可能性如何？可能性的大小通过置信度来表示。

    有了这些数据，商家就可以根据顾客的消费记录来为顾客推荐商品，比如我们在淘宝上搜索了一些商品之后，它总是会为我推荐一些相似的商品，说是”查看此商品的顾客也查看了如下商品”。这就是通过频繁项集与关联分析得到的，这有的时候非常有用，因为我们并不一定知道我们想要的商品的确切名字，往往是通过系统推荐的商品才找到了自己想要的。这种例子还有豆瓣读书与豆瓣电影，打开一部电影的介绍，网页中同样有喜欢这部电影的人也喜欢····的标题，并且推荐了一系列的电影给用户。

    除此之外，找频繁项集还可以用来发现某一类事物的共有特征。比如归类不同消费群体的消费清单，然后找出其中的频繁项集，并用频繁项集的元素来作为这一类消费群体的特征，以后就可以根据这些特征来推断消费者的身份。

    说到这里都还没提到Apriori算法，其实找频繁项集不用Apriori算法也一样可以完成，只不过效率很成问题。后面再做介绍，简而言之，该算法就是用来提升计算的效率的。

github部分

布置博客

之前以为github那部分完成以后，打开链接就能有基本的博客框架出现了，结果发现就只有index.html里面的那句“hello,world”！顿感眼前一黑，难道我要用自己只粗略浏览过的前端知识来完成整个博客？难以想象！还好一搜索，发现有很多搭建个人博客的简便方法。hexo就很不错。cnfeat这个教程讲的挺不错的，很详细，跟着一步步来，安装node.js,git我本来就有了，就不用安装了。之后安装hexo，对照着一步步配置。然后问题就来了—我输入hexo server这个命令，得到的结果告诉我hexo没有这个命令，搜索了也找不到原因，打开hexo的commands页面又有server命令，还以为是我的hexo版本太低，重装、更新都还是一样。还好在hexo文档的这个页面发现了原因

> Server was separated from the main module in Hexo 3. You have to install hexo-server before start using.

原来还要安装一遍，果然，安装完以后hexo server命令就可以用了。

之后还遇到一个问题，在准备deploy的时候。出现ERROR deployer not found github，还是到了hexo官方文档查看，可能是版本问题，在deploy下面的type不填github，填git就好。然后再执行npm install hexo-deployer-git —save。之后再hexo deploy就大功告成啦。

现在的问题是，markdown的语法我还是渣渣，还要学学才能愉快的写blog，哈哈
